The paper studies LLM routing: given a user query \(q\), choose between a strong but expensive model and a weaker, cheaper model to optimize the quality–cost trade-off.

Formally, let

\[

M {\text{strong}}, M {\text{weak}} : \mathcal Q \to \mathcal A

\]

be two language models, where \(M(q)\) denotes the answer produced by model \(M\) on query \(q\).

The routing problem is to learn a policy

\[

R:\mathcal Q \to \{M {\text{strong}}, M {\text{weak}}\}

\]

(or equivalently a router \(M R(q)\in\mathcal A\)) that:

- routes “easy” queries to the weak model to save cost,
- routes “hard” queries to the strong model to preserve quality,
- uses only one model call per query at inference time,
- ideally generalizes across domains and across model pairs without retraining.

This is a cost-sensitive binary classification / decision problem: for each query, infer whether the strong model is likely to outperform the weak one.
